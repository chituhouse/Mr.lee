# 飞书多维表格 AI 字段配置指南

本文档提供飞书多维表格 AI 字段捷径的配置模板，用于在各平台热搜表中实现自动化内容生成。

---

## 一、各平台热搜表字段设计

### 1.1 基础字段（已有）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 标题 | 单行文本 | 热搜话题标题 |
| 排名 | 数字 | 当前排名 |
| 热度 | 数字 | 热度数值 |
| 增长值 | 数字 | 相比上一轮的增长 |
| 日增长值 | 数字 | 相比今日首轮的增长 |
| 排名变化 | 数字 | 排名变化（正数=上升） |
| 状态 | 单选 | 新/上升/下降/持平 |
| 来源平台 | 单行文本 | 微博/抖音/头条/知乎/B站 |
| 摘要 | 多行文本 | 话题描述 |
| 链接 | URL | PC 端链接 |
| 移动端链接 | 单行文本 | 移动端链接 |
| 抓取时间 | 日期 | Unix 时间戳 |
| 采集批次 | 单行文本 | 批次标识（便于清理） |

### 1.2 AI 字段（需新增）

#### AI 字段 1：适配度初筛

**字段名**：`适配度（AI）`
**类型**：AI 字段（单选）
**可选值**：高 / 中 / 低

**Prompt 模板**：

```
【角色】老李选题助手

【老李人设】
40岁中年打工人
办公室老油条，乐观中带点无奈
擅长：职场吐槽、社会观察、谐音梗、反转幽默
风格：小人物视角解读大事件，笑中带泪

【适配度标准】
高：职场、民生、中年共鸣、有反转空间、全民热议
中：可以切入但需要改编
低：纯八卦、专业技术、敏感政治、小众圈层

【任务】
根据话题标题"{标题}"、摘要"{摘要}"、热度{热度}、状态"{状态}"，判断适配度。

【输出】
只输出：高 或 中 或 低
```

---

#### AI 字段 2：老李金句（初版）

**字段名**：`老李金句（AI）`
**类型**：AI 字段（多行文本）
**触发条件**：仅当 `适配度（AI）` = "高" 时生成

**Prompt 模板**：

```
【角色】老李金句创作师

【老李人设】
40岁中年打工人，办公室老油条
性格：乐观中带点无奈，犀利中透着温情
语言风格：接地气、有反转、打工人共鸣

【任务】
为话题"{标题}"（摘要：{摘要}）创作一句老李风格金句。

【要求】
1. 25-45岁打工人能共鸣
2. 有反转或自嘲
3. 15-30字
4. 不用网络烂梗

【输出】
只输出金句内容，不要标点符号和解释
```

### 1.3 确认快照字段（防刷新丢失）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 确认版适配度 | 单选 | 人工确认后的适配度（高/中/低） |
| 确认版金句 | 多行文本 | 人工确认后的金句（防 AI 刷新覆盖） |
| 已确认 | 复选框 | 勾选后表示内容已人工确认 |
| 推荐到选题池 | 复选框 | 勾选后触发转移到选题池 |

## 二、确认快照工作流

### 2.1 操作步骤

```
1. AI 字段自动生成适配度和金句
   ↓
2. 团队浏览 AI 生成结果，必要时手动调整
   ↓
3. 满意后：
   - 复制 AI 字段内容到"确认版"列
   - 勾选"已确认" ✓
   ↓
4. （可选）勾选"推荐到选题池" ✓
   ↓
5. 后续 AI 字段可以随意刷新，确认版内容不受影响
```

### 2.2 按钮字段配置（可选）

飞书支持按钮字段执行自动化。可添加按钮：

**按钮名称**：`确认并快照`
**触发动作**：
```
将 "适配度（AI）" 复制到 "确认版适配度"
将 "老李金句（AI）" 复制到 "确认版金句"
勾选 "已确认"
```

## 三、选题池表设计

### 3.1 字段列表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 来源平台 | 单选 | 微博/抖音/头条/知乎/B站 |
| 标题 | 单行文本 | 热搜标题 |
| 热度 | 数字 | - |
| 增长值 | 数字 | - |
| 适配度 | 单选 | 高（只接收高适配） |
| 推荐理由 | 多行文本 | - |
| 老李金句 | 多行文本 | 从热搜表转移过来的确认版金句 |
| 完整脚本（AI） | AI 字段 | 进入选题池后生成 3-5 句段子 |
| 完整脚本确认版 | 多行文本 | 人工确认后的脚本 |
| 预估时长 | 单行文本 | 如"30秒" |
| 链接 | URL | 回源链接 |
| 推荐时间 | 日期 | Unix 时间戳 |
| 是否采用 | 复选框 | 团队最终决定是否使用 |
| 源记录 | 关联字段（可选） | 关联回热搜表的原始记录 |

### 3.2 完整脚本 AI 字段配置

**字段名**：`完整脚本（AI）`
**类型**：AI 字段（多行文本）

**Prompt 模板**：

```
【角色】老李段子编剧

【老李人设】
40岁中年打工人，办公室老油条
性格：乐观中带点无奈，犀利中透着温情
擅长：职场吐槽、社会观察、谐音梗、反转幽默

【任务】
基于以下信息，创作一段完整的老李段子（用于30秒短视频）：

话题：{标题}
平台：{来源平台}
热度：{热度}
金句参考：{老李金句}

【要求】
1. 3-5句话
2. 总字数 80-120 字
3. 符合老李口吻（接地气、有共鸣、有反转）
4. 可以讲故事、自嘲、或吐槽
5. 结尾有金句或反转

【输出】
只输出段子内容，不要标题和解释
```

## 四、转移机制（热搜表 → 选题池）

### 方案 A：飞书自动化 + Webhook

```
触发条件：
  - 字段"推荐到选题池" = 已勾选 ✓
  - 且 字段"已确认" = 已勾选 ✓

执行动作：
  1. 发送 Webhook 到服务器
     URL: http://111.229.81.206:3000/trigger/transfer
     Body: { recordId, platform, title }

  2. 服务器收到 webhook 后：
     - 读取热搜表记录
     - 执行 Skill3 质量审核（可选）
     - 写入选题池
     - 触发 AI 字段生成完整脚本
```

### 方案 B：群聊 @机器人触发

```
用户在群里：
  @老李机器人 汇总今天的高适配选题

机器人执行：
  1. 读取各平台热搜表中"推荐到选题池"=✓的记录
  2. 批量转移到选题池
  3. 在群里回复汇总结果
```

## 五、Prompt 结构化改造示例

**旧格式（Markdown）**：
```
你是「老李动画」选题生成师。从热搜话题中筛选适合「老李」这个角色的选题。

## 老李人设
- 40岁中年男人，办公室打工人
- 性格：乐观中带点无奈，犀利中透着温情
...
```

**新格式（结构化）**：
```
【角色】老李动画选题生成师

【老李人设】
40岁中年打工人
办公室老油条，乐观中带点无奈
擅长：职场吐槽、社会观察、谐音梗、反转幽默

【适配度标准】
高：职场、民生、中年共鸣、有反转空间、全民热议
中：可以切入但需要改编
低：纯八卦、专业技术、敏感政治、小众圈层

【输出格式】
每条：标题 | 适配度 | 理由 | 金句
```

## 六、注意事项

1. **AI 字段刷新风险**
   - 全局刷新会覆盖所有历史内容
   - **务必使用确认快照列保护已确认的内容**
   - 单条刷新安全，全局刷新需谨慎

2. **增量查询优化**
   - 新增"采集批次"字段后，可按批次查询
   - 定期清理过期数据（如 1 个月前的非高适配记录）
   - 保留高适配记录作为历史参考

3. **成本控制**
   - AI 字段使用豆包模型（火山引擎算力）
   - 只对高适配生成金句（节省 70% 调用量）
   - 完整脚本仅在进入选题池后生成

4. **人机协作边界**
   - AI 负责：初筛、初稿生成
   - 人负责：最终决策、质量把关、风格调整
   - 通过"已确认"和"是否采用"两个复选框明确责任

**配置完成后，整个流程变为：**

```
数据采集（自动，cron 2h）
  ↓
AI 字段生成适配度和金句（自动，实时）
  ↓
团队浏览、调整、确认快照（人工，飞书表格）
  ↓
勾选"推荐到选题池" → 触发转移（自动 + 人工混合）
  ↓
进入选题池后 AI 生成完整脚本（自动）
  ↓
团队确认脚本、勾选"是否采用"（人工）
```
