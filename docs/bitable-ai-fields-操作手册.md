# 飞书 Bitable AI 字段配置操作手册

> 本文档提供逐步操作指南，用于在各平台热搜表中配置 AI 字段。
>
> **操作人**：用户（在飞书多维表格中手动配置）
> **配置对象**：微博/抖音/头条/知乎/B站 5 张热搜表

---

## 一、字段配置清单

### 必须新增的字段（按顺序）

| 序号 | 字段名 | 字段类型 | 说明 |
|------|--------|----------|------|
| 1 | 适配度（AI） | AI 字段 → 单选 | 豆包自动判断：高/中/低 |
| 2 | 文字梗挖掘（AI） | AI 字段 → 多行文本 | 豆包识别成语/谐音/字面理解空间 |
| 3 | 老李金句（AI） | AI 字段 → 多行文本 | 仅对高适配生成金句 |
| 4 | 确认版适配度 | 单选 | 人工确认后的适配度（快照） |
| 5 | 确认版金句 | 多行文本 | 人工确认后的金句（快照） |
| 6 | 已确认 | 复选框 | 勾选表示内容已人工确认 |
| 7 | 推荐到选题池 | 复选框 | 勾选触发转移到选题池 |

## 二、AI 字段配置详解

### 字段 1：适配度（AI）

**操作步骤：**

1. 在表格中点击「新增字段」→ 选择「AI 字段」
2. 字段名称：`适配度（AI）`
3. 输出类型：选择「单选」
4. 可选值设置：高、中、低（3 个选项）
5. Prompt 配置：复制下方 Prompt

**Prompt 模板：**

```
你是老李选题助手。老李是 30-40岁打工人，擅长文字游戏和反转笑话。

【判断标准】（满足 2 条以上 → 高适配度）
✅ 有文字梗空间（成语可改编、谐音词、可字面理解）
✅ 有反转潜力（期待 vs 现实的落差）
✅ 打工人共鸣（职场、生活、社交真实痛点）
✅ 话题接地气（非小众、非高端）

❌ 排除：严肃政治、负面社会事件、小众专业、过于沉重的悲剧

【输入信息】
标题：{标题}
摘要：{摘要}
热度：{热度}
状态：{状态}

【输出】
只输出一个字：高 或 中 或 低
```

**测试方法：**
配置完成后，在第一行手动填写测试数据：
- 标题：`不负众望的表现`
- 摘要：`某球员表现出色`
- 热度：`500000`
- 状态：`上升`

看 AI 字段是否输出"高"（因为"不负众望"可改编）。

### 字段 2：文字梗挖掘（AI）

**操作步骤：**

1. 新增字段 → AI 字段
2. 字段名称：`文字梗挖掘（AI）`
3. 输出类型：多行文本
4. Prompt 配置：

**Prompt 模板：**

```
你是文字梗挖掘专家。从标题中识别文字游戏潜力。

【识别类型】
1. 成语改编（如"不负众望" → 真的不负/辜负众人期望）
2. 谐音梗（如"走漏风声" → 口袋漏风了）
3. 字面理解（如"胖的人" → 被打成平）

【正面案例】
- "铁杵磨成针" → 用铁杵打人（字面理解）
- "无法企及的帅" → 帅到无法企鹅/站立（谐音）
- "省钱的秘诀" → 钱都没了，还省啥钱（反转）

【输入】
标题：{标题}

【输出】
如果有文字梗潜力，输出改编思路（20字以内）
如果没有，输出：无
```

**测试方法：**
第一行标题改为 `走漏风声`，看是否输出 `谐音梗：口袋漏风了`

---

### 字段 3：老李金句（AI）

**操作步骤：**

1. 新增字段 → AI 字段
2. 字段名称：`老李金句（AI）`
3. 输出类型：多行文本
4. **触发条件**：仅当 `适配度（AI）` = "高" 时生成（在 AI 字段配置里勾选条件）
5. Prompt 配置：

**Prompt 模板：**

```
你是老李本人，30-40岁打工人，说大白话。

【语言风格】
✅ 口语化："这事儿吧"、"你看啊"、"说实话"
✅ 反问句："你知道为啥吗？"
✅ 自嘲收尾："我给你们个段子吧"
❌ 避免：鸡汤式、文艺腔、段子手炫技

【正面案例】
- 话题"省钱秘诀" → 金句："钱都没了，还省啥钱？"
- 话题"极简风格" → 金句："小户型，那不叫极简，那叫塞不下！"
- 话题"胖的人" → 金句："胖的人嘛，被打成平了！"

【反面案例】（不要这样说）
❌ "虽然生活苦，但要保持微笑……"（鸡汤式）
❌ "岁月如诗，时光似水……"（文艺腔）
❌ "本来想做条咸鱼，没想到连盐都买不起……"（段子手炫技）

【输入信息】
标题：{标题}
摘要：{摘要}
文字梗：{文字梗挖掘（AI）}

【输出】
一句老李风格金句（15-30字），不要标点符号和解释
```

**测试方法：**
适配度为"高"的记录，应该自动生成金句。

## 四、工作流设计

### 流程图

```
数据采集（cron 2h）→ 写入热搜表
        ↓
AI 字段自动触发（实时）
        ↓
① 适配度（AI）生成
② 文字梗挖掘（AI）生成
③ 老李金句（AI）生成（仅高适配）
        ↓
团队浏览、调整
        ↓
满意后：复制 AI 内容到确认版字段
        ↓
勾选"已确认" ✓
        ↓
（可选）勾选"推荐到选题池" ✓
        ↓
触发转移（webhook / 群聊机器人）
```

### 确认快照操作

**手动方式：**
1. 查看 AI 字段生成内容
2. 如果满意：手动复制到确认版字段
3. 勾选"已确认"

**自动化方式（可选）：**
配置按钮字段「确认并快照」，点击后自动执行：
- 复制 `适配度（AI）` → `确认版适配度`
- 复制 `老李金句（AI）` → `确认版金句`
- 勾选 `已确认`

## 六、常见问题

### Q1：AI 字段没有自动生成？

**检查清单：**
1. 字段配置是否保存成功
2. Prompt 是否引用了正确的字段名（用 `{字段名}` 格式）
3. 触发条件是否设置正确（如"仅高适配时生成"）
4. 刷新页面重试

### Q2：豆包生成的金句质量不行？

**短期方案：**
- 在确认版字段里手动修改
- 后续用 Claude API 补充（Skill2 深度生成）

**长期方案：**
- 收集 10-20 个好案例，加入 Prompt 的正面案例库
- 调整 Prompt 描述（更具体）

### Q3：如何批量刷新 AI 字段？

**注意风险：**
- 单条刷新：安全，只重新生成当前记录
- 全局刷新：危险，会覆盖所有历史内容（包括已确认的）

**建议：**
- 只用单条刷新
- 已确认的内容用快照列保护
