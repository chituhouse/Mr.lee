# Skills 拆分草案（V1）

## 目标
将“热点采集 → 适配筛选 → 老李化改写 → 飞书入库 → 审核回流”拆成可执行技能模块，便于自动化与迭代。

## 技能清单

1) hotlist-weibo-collector
- 作用：采集微博热搜榜（优先官方/授权；无则合规第三方；再退到有头浏览器）
- 输出：标准化字段（平台、话题、热度、排名、来源链接、抓取时间）

2) hotlist-douyin-collector
- 作用：采集抖音热点（同样优先合规数据源）
- 输出：标准化字段（平台、话题、热度、排名、来源链接、抓取时间）

3) topic-fit-scoring
- 作用：判断“老李适配度（高/中/低）”并给出一句理由
- 规则：打工人/职场/中年情绪/生活尴尬优先；现实悲剧类降级或跳过

4) lao-li-writer
- 作用：将热点转译为老李风格内容
- 输出：金句 + 段子结构 + 30秒动画脚本

5) feishu-bitable-sync
- 作用：写入飞书多维表格；读取审核状态
- 输出：单元格内容、版本号、审核状态

6) review-loop
- 作用：若被标注“不合格”，读取原因并重新生成新版本
- 输出：版本号+1的新内容

## 数据字段建议（通用）
- 平台、话题、热度、排名、来源链接、抓取时间
- 老李适配度、高/中/低理由
- 内容输出（单元格模板）
- 审核状态（待审/通过/不合格）
- 审核原因
- 版本号

## 执行顺序（每日）
1) hotlist-weibo-collector
2) topic-fit-scoring
3) lao-li-writer
4) feishu-bitable-sync
5) review-loop（按需）

## MVP 最小实现路径
- 先跑微博：1 → 3 → 5
- 抖音第二阶段接入：2 → 3 → 5
- 小红书仅在合规数据源明确后接入
