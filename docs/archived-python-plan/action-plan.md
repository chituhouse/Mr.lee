# 老李 IP 选题引擎行动方案（规划版）

## 目标与范围
- 目标：建立“热点采集 → 适配筛选 → 老李化改写 → 飞书多维表格入库 → 审核回流”的自动化选题系统。
- 首期范围：微博优先，抖音重要但后续接入；小红书暂缓或仅做合规数据源。
- 产出形态：金句、段子结构、30 秒动画脚本（三类同格输出到多维表格单元格）。

## 我需要你提供的内容（分必需/可选）

### 必需（没有这些无法启动）
1) **老李人设基准（已根据现有信息预设）**
   - 人设描述（1 段话）：
     老李，40 岁中年打工人，见过世面但不端着，嘴碎爱抖包袱，喜欢谐音梗和无厘头反转；吐槽现实但不油腻，能自嘲也能共情，偏现实主义不鸡汤，以“打工人/职场/生活尴尬”为主要语境，风格接地气、有点丧但不消极。
   - 语言风格关键词：
     无厘头、接地气、谐音梗、自嘲、反转、职场视角、吐槽但不戾气。
   - 合规边界（红线/黄线/绿线）：
     - 红线：违法违规、造谣传谣、人身攻击、隐私曝光；煽动对立、群体歧视；低俗擦边、暴力血腥、未成年人不当内容。
     - 黄线：现实悲剧/事故类热点；指名道姓的个体/企业吐槽；容易引发地域/行业对立的话题。
     - 绿线：打工人情绪、职场人际、生活尴尬、消费焦虑、无厘头反转与自嘲类观察。

2) **输出格式模板（已根据现有信息预设）**
   - 金句格式（默认前缀“老李说：”）：
     老李说：「________。」（1 句，20–40 字）
   - 段子结构格式（铺垫/转折/包袱）：
     铺垫：________  
     转折：________  
     包袱：________  
   - 30 秒动画脚本格式（总时长 25–35 秒）：
     镜头1（5–8s）：画面/动作/环境 + 旁白或对白  
     镜头2（8–10s）：矛盾升级/情绪反应  
     镜头3（8–10s）：无厘头反转/谐音梗  
     结尾金句（3–5s）：老李说：「________。」  

3) **飞书多维表格信息**
   - Base ID
   - Table ID
   - 字段名是否按建议创建（平台、话题、热度、来源链接、抓取时间、适配度、理由、内容输出、审核状态、审核原因、版本号）

4) **审核回流规则**
   - 不合格是否新增一行（版本号+1），还是覆盖旧行
   - 不合格原因是必填还是选填
   - 通过后是否锁定、是否允许再改写

5) **MVP 数据路径选择（微博）**
   - A 申请官方渠道/授权数据
   - B 第三方合规数据服务
   - C 有头浏览器自动化（含人工处理验证码）

### 可选（有更好，没有也能先做）
1) **历史优质内容样本（10–30 条）**
   - 标注数据表现（点赞/评论/转发）
   - 用于建立“老李适配度”的隐性规则

2) **内容 KPI 优先级**
   - 播放 > 互动 > 评论，或其他优先级排序

3) **运营节奏**
   - 每天几点需要选题池
   - 每天需要多少条候选

4) **账号调性/风险偏好**
   - 更愿意保守稳妥 vs 更愿意激进博传播

## 行动计划（4 阶段）

### Phase 1：MVP 7 天跑通
- 目标：先有“稳定选题供给”，解决焦虑。
- 输出：每天 10–20 个候选选题 + 3 种内容形态。
- 方式：微博热搜榜自动化采集 + 老李适配筛选 + 改写入表。

交付物（文档/配置）：
- 老李人设系统 Prompt（正式版）
- 选题筛选 Prompt
- 输出格式模板
- 飞书多维表格字段定义与样例

### Phase 2：稳定化（2–4 周）
- 加入抖音热点
- 引入审核回流（不合格自动重写）
- 引入“热度变化、增长趋势”字段

交付物：
- 抖音采集方案
- 审核回流流程图

### Phase 3：数据与质量提升（1–2 个月）
- 多平台聚合对齐（微博+抖音+小红书）
- 引入历史数据复盘与风格学习
- 加入内容风险过滤器

### Phase 4：产品化（3 个月）
- 支持多个 IP/多个账号
- 多人协作与权限管理
- 指标仪表盘（点击/完播/互动）
